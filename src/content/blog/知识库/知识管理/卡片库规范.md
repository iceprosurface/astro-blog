## 概述

本文档定义了 Obsidian 卡片库的创建和维护规范，旨在保持卡片的原子性、可检索性和可连接性。

## 卡片类型定义

| 前缀 | 英文全称 | 中文名称 | Permalink 路径 | 标题前缀 | 说明 |
|------|---------|---------|----------------|---------|------|
| **CON** | Concept | 概念卡片 | `/card-library/concept/` | `## 核心概念` | 解释某个概念的定义、含义和特征 |
| **CMP** | Comparison | 对比卡片 | `/card-library/comparison/` | `## 核心差异` | 对比两个或多个事物的异同、优劣 |
| **IDEA** | Idea | 灵感卡片 | `/card-library/idea/` | `## 核心概念` | 创意、灵感、未成型的想法 |
| **INSIGHT** | Insight | 洞见卡片 | `/card-library/insight/` | `## 核心洞见` / `## 核心观点` | 深入的观点、反直觉的发现、深度思考 |
| **METHOD** | Method | 方法卡片 | `/card-library/method/` | `## 核心方法` | 具体操作步骤、方法论、技巧 |
| **PRACTICE** | Practice | 实践卡片 | `/card-library/practice/` | `## 核心实践` | 个人或团队的实际案例、经验总结、应用场景 |
| **CASE** | Case | 案例卡片 | `/card-library/case/` | `## 案例背景` | 外部行业案例、商业分析、事件记录、参考案例 |

## 文件命名规范

### 基本格式

```
TYPE_主题.md
```

### 示例

- `CON_卡片笔记法.md`
- `CMP_中美社会制度对比.md`
- `INSIGHT_AI时代的工程护城河.md`
- `METHOD_API预加载策略.md`
- `PRACTICE_Tailwind样式提取.md`

### 命名规则

1. **TYPE**：必须使用上述表格中定义的前缀（大写）
2. **主题**：简洁、清晰的中文名称，避免特殊字符
3. **分隔符**：使用下划线 `_` 分隔 TYPE 和主题

## Permalink 规范

### 基本格式

```
/card-library/{type}/{name}/
```

### 参数说明

- **type**：卡片类型的小写形式（concept, comparison, insight, idea, method, practice, case）
- **name**：英文标识符，使用 kebab-case（小写字母和连字符）

### 示例

| 文件名 | Permalink |
|--------|-----------|
| `CON_美国斩杀线.md` | `/card-library/concept/us-killing-line/` |
| `CMP_中美社会制度对比.md` | `/card-library/comparison/us-china-social-systems/` |
| `INSIGHT_AI时代的工程护城河.md` | `/card-library/insight/engineering-moats-in-ai-era/` |

## Frontmatter 规范

### 必填字段

```yaml
---
title: 卡片标题
date: 2026-01-22T00:00:00+08:00
updated: 2026-01-22T00:00:00+08:00
permalink: /card-library/{type}/{name}/
tags:
  - 标签1
  - 标签2
ccby: false
draft: false
comments: true
no-rss: true
---
```

### 字段说明

| 字段 | 说明 |
|------|------|
| **title** | 卡片标题（必须与文件名中的主题一致） |
| **date** | 创建日期（ISO 8601 格式） |
| **updated** | 最后更新日期（ISO 8601 格式） |
| **permalink** | 遵循 Permalink 规范 |
| **tags** | 标签列表，用于分类和检索 |
| **ccby** | 是否使用 CC BY 协议（卡片内容通常为 false，因为内容通常有不确定性，先保留版权和传播许可，等整理成文后开放） |
| **draft** | 是否为草稿（发布时设置为 false） |
| **comments** | 是否开启评论 |
| **no-rss** | 是否不包含在 RSS 中（**AI 助手规则**：默认设为 `true`，需手动开启 RSS 才会设为 `false`） |

## 链接规范

### 卡片间链接（必须使用相对路径）

**正确**：
```markdown
- [中美社会制度对比](./CMP_中美社会制度对比.md)
- [美国斩杀线](./CON_美国斩杀线.md)
```

**错误**：
```markdown
- [中美社会制度对比](/知识库/卡片库/CMP_中美社会制度对比.md)
```

### 原则

1. **仅链接卡片库内部**：只链接到 `public/知识库/卡片库/` 下的卡片
2. **使用相对路径**：以 `./` 开头
3. **保持文件名一致**：链接的文件名必须与实际文件名完全一致
4. **删除私有目录链接**：不链接到 **`01-xxx/`** 等私有目录

### 正文中的链接

正文中需要引用其他卡片时，也使用相对路径链接，或直接使用文字说明：

```markdown
这一观点与 [AI 时代的工程护城河](./INSIGHT_AI时代的工程护城河.md) 中的思想一致。

或：

这一观点与 "AI 时代的工程护城河" 中的思想一致。
```

## 卡片核心原则

### 原子性
- 每张卡片只包含一个完整的知识点
- 避免将多个主题混在同一张卡片中

### 自主性
- 卡片内容独立可理解
- 不依赖其他卡片也能被阅读和理解

### 连接性
- 通过链接形成知识网络
- 鼓励发现卡片间的意外关联

### 可检索性
- 使用清晰的标题和标签
- 遵循命名规范便于查找

## 注意事项

1. **避免链接到私有 vault**
   - 卡片库内容会同步到公开博客
   - 仅包含公开内容，不引用 `01-xxx/` 等私有目录

2. **保持文件名与 permalink 对应**
    - permalink 应该是语义化的英文标识符

3. **标签的使用**
   - 每张卡片至少有 1-3 个标签
   - 标签应反映卡片的主题和类型

4. **更新日期**
   - 每次修改卡片内容时，更新 `updated` 字段
